# T02: Added /teams dashboard route with role badges, empty state, team cards, and /teams/$teamId stub; index redirects to /teams and sidebar reorders Teams ahead of Items

**Added /teams dashboard route with role badges, empty state, team cards, and /teams/$teamId stub; index redirects to /teams and sidebar reorders Teams ahead of Items**

## What Happened

Built the Teams Dashboard as the post-login landing page. The `/teams` route fetches `TeamsService.readTeams()` via `useSuspenseQuery` with a `['teams']` query key (matches the S04 slice-plan cache-key scheme) and casts the generic `{[key: string]: unknown}` SDK response to `{data: TeamWithRole[], count: number}` — the Python handler returns `dict[str, Any]` which openapi-ts widens to unknown, but the runtime envelope is stable and validated by backend tests.

Each team renders as a Card acting as a `Link` to `/teams/$teamId`, with a `data-testid="team-card"` handle, a `role-badge` (default variant for admin, secondary for member) and an outline `Personal` badge when `is_personal`. Sort is server-side (backend orders by `created_at desc`), so no client re-sort. Cards are single column on small screens and 2-col on `md:` for the 375px → tablet responsive brief. Name truncates with `max-w-[60%]` on a single line alongside the badges.

The empty state (a card with `Users` icon, `No teams yet` copy, and a disabled `data-testid="create-team-button"` — T03 will wire the click handler) renders only when `data.data.length === 0`. The `Welcome back, nice to see you again!!!` copy lives in the page header so the existing Playwright `login.spec.ts` text assertion keeps passing.

`/_layout/index.tsx` switched to a `beforeLoad` that throws `redirect({ to: "/teams", replace: true })` — the simplest form the plan prescribed. As a necessary local adaptation I updated the six `page.waitForURL("/")` calls across `tests/login.spec.ts` (×3), `tests/auth.setup.ts` (×1), and `tests/utils/user.ts` (×1) to `waitForURL("/teams")` so the Playwright flow still settles after the redirect. The slice plan called out the TEXT assertion but not the URL one — fixing the URL waits is the same routing change and belongs in this task.

The sidebar now starts with `Teams` (Users icon) → `/teams`, followed by `Items`. I removed the old `Home → /` entry entirely rather than keep a duplicate link to the same page. Admin tail still appended for `system_admin`.

`teams.$teamId.tsx` is a placeholder route so card navigation doesn't 404 before T04; it reads the param with `Route.useParams()` and shows a short "lands here in T04" note with `data-testid="team-detail-placeholder"`.

Route tree regenerated by the Vite TanStack Router plugin during `bun run build` — the generated file now imports `LayoutTeamsRouteImport` and `LayoutTeamsTeamIdRouteImport` and threads them through `LayoutRouteChildren`.
