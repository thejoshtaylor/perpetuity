# T03: Add /admin/teams paginated list page with sibling-form route, AdminTeamsColumns table, and sidebar entry for system admins

**Add /admin/teams paginated list page with sibling-form route, AdminTeamsColumns table, and sidebar entry for system admins**

## What Happened

Built the system-admin teams list page that consumes T01's `GET /admin/teams` via T02's regenerated `AdminService.readAllTeams`.

**Files added/changed:**
- `frontend/src/routes/_layout/admin_.teams.tsx` — sibling-form file route at URL `/admin/teams`, gated with `beforeLoad: requireSystemAdmin` from T02. Uses `useSuspenseQuery` against `AdminService.readAllTeams({ skip, limit })` with `validateSearch` parsing `skip` (default 0) and `limit` (default 20) from URL search params, coerced to non-negative integers. Renders heading 'All Teams' + subtitle 'System admin: every team in the workspace.' over the shared `DataTable`. Pagination uses Prev/Next buttons that call `useNavigate({ from: Route.fullPath })` to update the URL search; Prev disabled when `skip <= 0`, Next disabled when `data.data.length < limit`. Empty state (only when skip==0) renders a `<Card>` with the spec'd 'No teams in the system yet.' copy. Suspense fallback is a 5-row skeleton table.
- `frontend/src/components/Admin/AdminTeamsColumns.tsx` — column defs for the admin teams table (`AdminTeamRow` type matching backend `TeamPublic`: id, name, slug, is_personal, created_at). Columns: Name, Slug (mono-font), Personal? (badge), Created (locale date), and an actions column with a 'View members' link to `/admin/teams/$teamId`.
- `frontend/src/components/Sidebar/AppSidebar.tsx` — added `{ icon: Shield, title: 'All Teams', path: '/admin/teams' }` to the system-admin sidebar items, alongside the existing 'Admin' entry.
- `frontend/src/routeTree.gen.ts` — regenerated by the Vite router plugin to register the new route.

**Key decisions:**
1. **Sibling-form route file (`admin_.teams.tsx`, not `admin.teams.tsx`)** — TanStack Router's dot-prefix convention makes `admin.teams.tsx` a CHILD of `admin.tsx`, but `admin.tsx` has no `<Outlet />` so the child would never render (MEM048/MEM053). Trailing-underscore opt-out keeps the URL `/admin/teams` while making it a sibling — parent UI is fully replaced. Matched the pattern already used by `teams_.$teamId.tsx`. Route id is `/_layout/admin_/teams` (the underscore is preserved in the routeTree).
2. **Plain anchor for 'View members' link** — TanStack Router's `<Link to=...>` is type-validated against the route tree, but `/admin/teams/$teamId` doesn't exist yet (T04 will create it). Using a plain `<a href={...}>` avoids breaking T03's typecheck without forcing a forward dependency on T04.
3. **`as unknown as AdminTeamsEnvelope` cast** — backend's `read_all_teams` returns `dict[str, Any]`, so the OpenAPI generator types `AdminReadAllTeamsResponse` as `{[k:string]: unknown}`. Mirrored the cast pattern already used by `teams.tsx` (`as unknown as ReadTeamsEnvelope`) — kept the cast local to the queryFn so callsites stay strongly typed.
4. **`validateSearch` over hand-rolled parsing** — using TanStack Router's `validateSearch` lets `Route.useSearch()` return `{skip, limit}` strongly typed, and `useNavigate({ search })` does the type-checked URL update.
5. **Empty state only when `skip === 0`** — paginating past the last page should not render the empty state (it's not "no teams in the system" — it's "you scrolled too far").

**Gotcha captured (MEM060):** `bun run build` runs `tsc && vite build`; `tsc` fails first when adding a new file route because the Vite plugin regenerates `routeTree.gen.ts` only during the vite step. Resolution: run `bunx vite build` once to seed the route tree, then `bun run build` passes. Documented in memory for future route additions.
