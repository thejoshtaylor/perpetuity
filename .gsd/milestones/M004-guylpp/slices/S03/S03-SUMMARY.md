# S03: Team-mirror container + lifecycle (spin-up, reap, always-on toggle)

**Per-team git mirror containers with idempotent ensure, idle reaper, and always-on admin toggle â€” the credential-free transport substrate every M004 git-op sits on.**

## What Happened

S03 lays the integration centerpiece for M004's two-hop clone story: every team gets its own long-lived git daemon container that user containers can dial by compose-DNS (`team-mirror-<first8-team>:9418`) without seeing or holding any GitHub credentials. The slice spanned four tasks across backend (schema + admin validator + team-admin PATCH endpoint), orchestrator (ensure/reap module + reaper loop + HTTP routes), and a live-stack e2e proving the contract end-to-end.

**T01 â€” schema + registry foundation.** Created `team_mirror_volumes` (UUID PK, UNIQUE team_id FKâ†’team CASCADE, UNIQUE volume_path, nullable container_id + last_started_at + last_idle_at, always_on BOOLEAN DEFAULT FALSE) via alembic revision `s06c`. Added matching `TeamMirrorVolume`/`TeamMirrorVolumePublic`/`TeamMirrorPatch` SQLModels and registered `mirror_idle_timeout_seconds` in admin `_VALIDATORS` with a stricter [60, 86400] floor (sub-60s would weaponize the reaper into per-tick teardown). 16 backend tests cover migration upgrade-shape, FK CASCADE, server-default identity, downgradeâ†’re-upgrade byte-equivalence, and the 10 validator boundary cases.

**T02 â€” orchestrator integration centerpiece.** Three new modules: `team_mirror.py` (idempotent `ensure_team_mirror`: find-or-insert row â†’ label-list for existing container â†’ on miss `create_or_replace`+start, on 409 race fall back to filter-list lookup; pure helper `is_row_reapable(row, idle_timeout, now)` returns `(reapable, reason)` for the boundary table; `reap_team_mirror` stops+force-deletes the container with 5s timeout, NULLs container_id, NEVER touches the named docker volume), `team_mirror_reaper.py` (structurally separate asyncio.Task from the user-session reaper; per-tick resolves `mirror_idle_timeout_seconds` from system_settings; per-row failures (DockerError/OSError/WorkspaceVolumeStoreUnavailable) are log-and-continue so one bad row can't shadow other reaps; broad-except wrapper logs WARNING `team_mirror_reaper_tick_failed`), `routes_team_mirror.py` (POST /v1/teams/{id}/mirror/{ensure,reap}, gated by existing SharedSecretMiddleware). Wired into `main.py` lifespan with mirror reaper teardown FIRST (before user-session reaper, registry, pg, docker â€” both reapers read pg+docker, so reverse ordering surfaces as `team_mirror_reaper_tick_failed` warnings on every shutdown). Container config: same workspace image as user containers (D022), labels `team_id` + `perpetuity.managed=true` + `perpetuity.team_mirror=true`, name `team-mirror-<first8>`, cmd `git daemon --base-path=/repos --export-all --reuseaddr --enable=receive-pack --port=9418 --listen=0.0.0.0` (D023), named volume `perpetuity-team-mirror-<first8>` mounted at /repos, HostConfig.NetworkMode=`perpetuity_default` so user containers can resolve us by DNS. 29 unit tests cover the pure helpers, the is_row_reapable boundary table (5 cases), ensure/reap with an in-memory _FakePool+_FakeDocker harness (cold-start, warm-reuse with last_idle_at bump, 409 race, docker-unavailable, reap-happy, reap-no-container, reap-404 race), and the reaper loop (tick skip-on-always_on, skip-on-recent-activity, reap-on-idle, pg-unreachable propagation, cancel-clean-exit, docker-handle-None tick-skip).

**T03 â€” backend admin toggle.** Thin team-admin endpoint `PATCH /api/v1/teams/{id}/mirror` flips `always_on` on the row. Backend never calls the orchestrator â€” the toggle is pure DB state biasing the next reaper tick. On first PATCH for a team that has never spun up a mirror, auto-inserts a row with `volume_path='pending:<team_id>'` so admins can pre-toggle; the orchestrator's ensure path replaces the placeholder atomically on cold-start (UNIQUE on team_id forces UPDATE-not-INSERT, no race). Reused `_assert_caller_is_team_admin` (404 missing team â†’ 403 not-admin ordering, MEM047) and post-commit `session.refresh(row)` (MEM035). INFO log line `team_mirror_always_on_toggled team_id=<uuid> actor_id=<uuid> always_on=<bool> created_row=<bool>`. 9 tests cover auto-create on first PATCH, idempotent same-value-twice (single-row DB assertion), non-admin/non-member 403, missing team 404, missing field 422, non-bool 'maybe' 422 (pydantic v2 lax-bool would coerce 'yes' â†’ True silently â€” captured as MEM272), invalid uuid path 422.

**T04 â€” live-stack e2e.** Single `@pytest.mark.e2e @pytest.mark.serial` test in `backend/tests/integration/test_m004_s03_team_mirror_e2e.py` walks scenarios Aâ€“E end-to-end against compose db + ephemeral orchestrator (MIRROR_REAPER_INTERVAL_SECONDS=1, mirror_idle_timeout_seconds=60 â€” the validator floor â€” combined with manual back-dating of `last_idle_at`) + sibling backend + sibling alpine/git for the transport proof. A: cold-start ensure returns {container_id, network_addr, reused:false}; row inserted, container running with the expected labels, `team_mirror_started` log fires. B: second ensure returns same container_id with reused:true; exactly one container; `team_mirror_reused` log fires. C: `docker exec <mirror> git init --bare /repos/test.git` drops a fixture; alpine/git sibling on `perpetuity_default` runs `git clone git://team-mirror-<first8>:9418/test.git` (exit 0, .git/HEAD exists). D: PATCH always_on=true; back-date last_idle_at by 120s; reaper logs `team_mirror_reap_skipped team_id=<uuid> reason=always_on` and the container stays running. E: PATCH always_on=false; reaper logs `team_mirror_reaped team_id=<uuid> ... reason=idle`, container is gone, container_id is NULL but volume_path persists. Final structural sweep asserts the six required log markers all appeared and `team_mirror_reaper_tick_failed` did NOT. 16.92s wall-clock against a warm stack.

**One real bug found and fixed during T04 verification.** First draft asserted on the bare `team_mirror_reap_skipped` token â€” but the reaper had already fired one tick BEFORE the test's back-date logging `reason=recent_activity`, and a shortcut-on-first-hit log waiter accepted that line as if it were the always_on skip. Tightened to match on the full `team_mirror_reap_skipped team_id=<uuid> reason=always_on` substring (captured as MEM271).

**What S04 inherits.** Compose-DNS alias resolves `team-mirror-<first8>:9418` from any sibling on `perpetuity_default`. `ensure_team_mirror` returns `{container_id, network_addr}` ready to be the source of `git clone git://...` from user containers. `is_row_reapable` is the durable contract for any future per-team timeout policy. Mirror containers receive a fresh installation token in env vars on auto-push (S04 wires this via `git push origin --all --tags` from inside the mirror container â€” the credentials never enter the row, never enter logs, never enter user containers). The slice's named-volume durability invariant â€” reap NEVER removes the volume â€” is what makes S04's clone-once-then-mirror-many-times shape correct.

## Known Limitations

The conftest does not delete TeamMirrorVolume rows between unit-test runs, so accumulated rows persist across local test sessions; tests use unique team UUIDs so this is not a correctness issue, just a slow leak in a long-lived test database. The team CASCADE FK would clean these up on team delete, but the conftest doesn't delete teams either. Pre-existing flake in `tests/unit/test_github_tokens.py::test_get_installation_token_cache_miss_setex_ttl` (TTL boundary 3000 vs 2999 due to fakeredis clock-tick rounding) reproduces on main without S03 changes â€” unrelated to S03, out of scope, captured for S02 followup. The slice does not exercise concurrent-from-different-orchestrators ensure (we run one orchestrator); if a future deployment scales orchestrator horizontally, the UNIQUE on team_id will serialize at the DB but the docker create-or-replace path would need an external lock (out of scope until M-series scaling).
