# S03: Workflow definition CRUD + Celery run engine + button trigger

**Goal:** Land the workflow definition CRUD + form-field trigger + multi-step run engine + cross-step `{prev.stdout}` / `{form.<field>}` substitution + scope dispatch (user / team_specific / round_robin) + cancellation-between-steps + container-acquisition retry on top of S02's executor + run/step record schema, so a team admin can build, save, fire, and stop arbitrary workflows from the dashboard.
**Demo:** Team admin opens /workflows, creates 'lint and report' with trigger=button + form field `branch:string`, steps = `[git checkout {branch} (target=user_workspace), npm install (target=user_workspace), npm run lint (target=user_workspace), claude -p 'summarize lint output: {prev.stdout}' (target=user_workspace)]`, scope=user, saves. Workflow appears as a custom button in the dashboard. User clicks it, fills `branch=main`, hits Submit. Run page shows steps flip pending → running → succeeded/failed in real time via 1.5s polling. Final Claude step receives the prior step's stdout via `{prev.stdout}` substitution. Every step record persists snapshot + stdout + stderr + exit + duration. Cancel button between steps actually stops further execution.
