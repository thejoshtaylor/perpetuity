# Perpetuity

## What This Is

A collaborative developer workspace platform built on FastAPI + React. Teams of developers work on shared projects, but each user gets their own isolated Docker container per team they belong to — independent filesystem per user-team pair. Projects are team-scoped and tied to GitHub repositories; when a user starts working on a project, the repo folder is copied (not shared) into their team workspace container. AI coding assistants (Claude Code CLI, Codex CLI) run inside containers with per-team credentials. Workflows are Celery-backed trigger/step pipelines with full run history. The frontend is a PWA with mobile-first design and Grok STT voice input throughout.

## Core Value

A user can open their team workspace in a browser (or install as PWA), get a terminal in their isolated container, pull a GitHub project into it, and trigger AI-assisted coding workflows — all from one place, on any device.

## Architecture / Key Patterns

- **Backend:** FastAPI, SQLModel, PostgreSQL, Alembic migrations, Celery (async task queue)
- **Frontend:** React, TypeScript, Vite, React Query, Zustand, TanStack Router; PWA via vite-plugin-pwa injectManifest
- **Auth:** httpOnly cookie sessions; FE `OpenAPI.WITH_CREDENTIALS=true`, no localStorage tokens; auth-state truth = `useQuery(['currentUser'])` populated by route-guard `ensureQueryData`; logout calls `AuthService.logout()` because httpOnly cookies cannot be cleared client-side
- **Orchestrator service (M002+):** Top-level package owning the Docker socket (D005 — backend never gets it). Provisions per-(user, team) containers (label-scoped, name `perpetuity-ws-<first8-team>`, mem_limit=2g, pids_limit=512, nano_cpus=1.0, RestartPolicy=no). Runs `privileged: true` so loopback-ext4 LOOP_SET_FD succeeds on Docker Desktop / linuxkit (MEM136); privilege bounded to orchestrator only — workspace containers stay unprivileged.
